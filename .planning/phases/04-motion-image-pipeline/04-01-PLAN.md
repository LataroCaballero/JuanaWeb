2. Create `src/scripts/scroll-animations.ts` with this exact content:

```typescript
import { gsap } from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';

gsap.registerPlugin(ScrollTrigger);

const mm = gsap.matchMedia();

mm.add(
  {
    noPreference: '(prefers-reduced-motion: no-preference)',
    reduceMotion: '(prefers-reduced-motion: reduce)',
  },
  (context) => {
    const { reduceMotion } = context.conditions as {
      reduceMotion: boolean;
      noPreference: boolean;
    };

    if (reduceMotion) {
      gsap.set('#historia', { opacity: 1, y: 0 });
      gsap.set('#ubicaciones', { opacity: 1, y: 0 });
      return;
    }

    gsap.from('#historia', {
      opacity: 0,
      y: 60,
      duration: 0.9,
      ease: 'power2.out',
      scrollTrigger: {
        trigger: '#historia',
        start: 'top 85%',
        once: true,
      },
    });

    gsap.from('#ubicaciones', {
      opacity: 0,
      y: 60,
      duration: 0.9,
      ease: 'power2.out',
      scrollTrigger: {
        trigger: '#ubicaciones',
        start: 'top 85%',
        once: true,
      },
    });
  }
);
```

Key decisions:
- `once: true` on ScrollTrigger — single-fire reveal, not repeating
- `y: 60` — 60px upward slide with fade, per research recommendation
- `duration: 0.9` with `power2.out` easing — cinematic but not slow
- Targets ONLY `#historia` and `#ubicaciones` section wrappers — NOT inner card elements (avoids conflict with CSS grayscale hover on ubicaciones cards)
- `gsap.set()` in reduceMotion branch ensures elements are visible without animation
- `gsap.registerPlugin(ScrollTrigger)` called immediately after import to prevent tree-shaking
