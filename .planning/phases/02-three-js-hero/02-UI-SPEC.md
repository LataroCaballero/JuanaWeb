## Spacing Scale

All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight metadata gaps |
| sm | 8px | Compact inline spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section internal padding |
| xl | 32px | Layout column gaps, hero vertical padding mobile |
| 2xl | 48px | Hero vertical padding desktop, canvas overlap offset |
| 3xl | 64px | Hero section top padding on large screens |

Exceptions:

- Desktop canvas overlap: `margin-left: -48px` — 3D logo invades h1 column by 48px. Deliberate grid-break per Organic Brutalism. Source: CONTEXT.md D-08.
- Mobile canvas overlap: `margin-top: -24px` — canvas rises 24px over h1 bottom edge. Source: CONTEXT.md D-09.
- Canvas z-index: must be below `9999` (grain overlay `body::after`). Use `z-index: 100`. Source: `global.css` grain z-index.

## Color

| Role | Hex | CSS Variable | Usage |
|------|-----|--------------|-------|
| Dominant (60%) | `#131313` | `var(--color-surface)` | Page background, hero background, canvas clear color |
| Secondary (30%) | `#1a1a1a` / `#353534` | `var(--color-surface-low)` / `var(--color-surface-high)` | Depth layering if additional containers needed |
| Accent (10%) | `#0055ff` | `var(--color-electric)` | 3D logo MeshStandardMaterial color — ONLY element |
| On-surface | `#e5e2e1` | `var(--color-on-surface)` | H1 text color |
| Error / semantic | `#ffb4ab` | `var(--color-error)` | Not used in this phase |

Accent reserved for: **the Three.js 3D logo material** (`MeshStandardMaterial { color: #0055ff }`) exclusively. Electric Blue does NOT appear as text color, button color, border, or background anywhere else in the hero.

Source: CONTEXT.md D-03, `src/styles/global.css` @theme tokens.

### Three.js Material Spec

```
MeshStandardMaterial:
  color:     #0055ff   (var --color-electric)
  metalness: 0.7
  roughness: 0.2
```

Effect: chrome-blue metallic sheen. Premium, consistent with brand palette.

### Lighting Spec (Claude's Discretion — resolved)

```
AmbientLight:
  color:     #ffffff
  intensity: 0.4

DirectionalLight:
  color:     #b6c4ff   (var --color-electric-light — warm metallic tint)
  intensity: 1.2
  position:  [5, 8, 5]   (top-right-front — activates metallic specular on upper face)
```

Rationale: AmbientLight at 0.4 keeps shadow areas visible without flattening the metallic effect. DirectionalLight using `#b6c4ff` at top-right creates a brand-coherent chrome sheen — electric light color in the highlights, deep surface color in the shadows. No fill light — the hard shadow on the left side reinforces the brutalist aesthetic.

### Additional Visual Rules

- No 1px borders anywhere. Per DESIGN.md "No-Line Rule" — separation through color shift only.
- Zero roundedness: `border-radius: 0px` on all elements. No exceptions.
- No generic grey shadows. Ambient glow only if needed: `30px blur, 8% opacity, #0055ff tint`. Source: DESIGN.md §4.
- Canvas renderer: `renderer.setClearColor(0x000000, 0)` — transparent background inherits `#131313` from page.

## Canvas & Geometry Spec

### SVG Extrusion (CONTEXT.md D-01, D-05)

| Property | Value |
|----------|-------|
| SVG file | `src/assets/smiley-logo.svg` |
| SVG structure | `<path>` elements only — no `<use>`, no `<symbol>` |
| SVG contents | Outer circle + two arrow-shaped eyes |
| Extrusion depth | 10 Three.js scene units (mid-range of D-05: 8–12) |
| Loader | `THREE.SVGLoader` from `three/examples/jsm/loaders/SVGLoader.js` |
| Geometry class | `THREE.ExtrudeGeometry` |

### Canvas Dimensions (Claude's Discretion — resolved)

| Breakpoint | Canvas Size | CSS Declaration |
|------------|-------------|-----------------|
| Desktop (≥ 768px) | 45vmin × 45vmin | `width: 45vmin; aspect-ratio: 1 / 1` |
| Mobile (< 768px) | 70vw × 70vw | `width: 70vw; aspect-ratio: 1 / 1` |

`aspect-ratio: 1 / 1` is set on the canvas wrapper `<div>` in CSS before any JS loads. This reserves the layout space and satisfies HERO-03 (no CLS).

### Hero Layout

| Breakpoint | Layout | Overlap |
|------------|--------|---------|
| Desktop (≥ 768px) | CSS Grid 2-column: h1 left (7 cols), canvas right (5 cols) | `margin-left: -48px` on canvas |
| Mobile (< 768px) | Flex column: h1 top, canvas bottom | `margin-top: -24px` on canvas |

The overlap creates the Organic Brutalism effect — elements breaking container bounds per CONTEXT.md D-08/D-09.

### Idle Animation (CONTEXT.md D-06, Claude's Discretion for values)

| Property | Value |
|----------|-------|
| Y-axis rotation | `mesh.rotation.y += 0.003` rad/frame |
| Float position | `mesh.position.y = Math.sin(elapsed * 0.8) * 0.05` |
| Float amplitude | 0.05 scene units |
| Float frequency | 0.8 rad/s — slow, breath-like (~7.8s full cycle) |
| Entry animation | None — mesh appears at rest and begins floating immediately (CONTEXT.md D-07) |
| prefers-reduced-motion | Animation loop does not start; mesh renders static |

## Interaction Contract

| Trigger | Response |
|---------|----------|
| Page load, hero not in viewport | Three.js does NOT initialize. Canvas wrapper shows reserved space via `aspect-ratio` CSS. |
| Hero enters viewport (IntersectionObserver threshold: 0.1) | Dynamic import of Three.js island. Renderer initializes. Animation loop starts. |
| Idle (hero in viewport) | Float + Y-rotation loop runs via `requestAnimationFrame`. |
| Hero leaves viewport | `cancelAnimationFrame` — loop pauses. GPU work stops. |
| `prefers-reduced-motion: reduce` | `window.matchMedia` check on init. Loop never starts. Mesh renders one frame only. |
| Page unload / component teardown | `geometry.dispose()`, `material.dispose()`, `renderer.dispose()`, `renderer.forceContextLoss()`. No GPU memory leak. (HERO-04) |
| WebGL unavailable / SVGLoader failure | Silent failure. Canvas wrapper collapses or stays blank. H1 headline remains fully visible. No user-facing error. |

Source: CONTEXT.md D-12, D-13, D-14. REQUIREMENTS.md HERO-01, HERO-03, HERO-04.

No hover interactions on the canvas. No click events on the 3D logo. The canvas is `aria-hidden="true"` — purely decorative.
