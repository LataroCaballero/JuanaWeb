## Decisions Made

- **@tailwindcss/vite over @astrojs/tailwind:** The official Astro integration silently installs Tailwind v3, breaking `@theme` syntax. Manual Vite plugin is the v4 path.
- **Space Grotesk weight 700 only:** @fontsource/space-grotesk max weight is 700 (no 900 exists). The CSS `font-weight: 900` can still reference it and the browser will use 700 as closest match.
- **grain.webp in public/assets/:** CSS string references (`url('/assets/grain.webp')`) cannot resolve files from `src/assets/` because Vite hashes those paths. `public/` is served verbatim.
- **Component tokens in :root not @theme:** `--nav-blur`, `--glow-blur` etc. don't map to Tailwind utility namespaces. Placing them in `@theme` would either be ignored or generate nonsensical classes.
- **oxide native binding in optionalDependencies:** Node 18 has a known npm bug with optional deps causing `@tailwindcss/oxide-darwin-arm64` to not install on first `npm install`. Fixed by explicitly adding all platform variants to `optionalDependencies`.
