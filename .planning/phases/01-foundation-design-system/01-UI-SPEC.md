## Spacing Scale

Standard 8-point scale. Tailwind v4 default spacing utilities cover all these values without custom token declarations.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight metadata `text-[0.7rem]` padding |
| sm | 8px | Compact element padding (tag padding `px-2 py-1`) |
| md | 16px | Default element spacing, nav horizontal padding |
| lg | 24px | Section padding horizontal (`px-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing (hero padding `py-24` = 96px, nearest 8-pt = 64px) |

**Exceptions:**
- Hero section vertical padding: `py-24` (96px) — design-driven, retains this value from reference HTML
- Body grain overlay `z-index: 9999` — not a spacing value, noted for implementation clarity
- Touch-target minimum: 44px height for any interactive element (accessibility floor — applies to `<button>` and `<a>` with padding)

**Source:** DESIGN.md §Cards & Lists mentions `spacing-8` (2.75rem) as list separator; reference HTML uses `px-6 py-4`, `px-8 py-3`, `py-24`, `py-32` as recurring values. All map to or are compatible with 8-point multiples.

## Typography

Two font families. Three functional roles declared for Phase 1 (display, label-sm accent, body). Four sizes specified.

| Role | Font Family | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------------|------|--------|-------------|----------------|-------|
| Display / Hero | Space Grotesk | `clamp(6rem, 10vw, 10rem)` — `text-[10rem]` on md+ | 700 (bold / black CSS target) | 0.85 (`leading-[0.85]`) | `-0.05em` (tight) | `<h1>` hero statement "COFFEE ON YOUR WAY" |
| Heading | Space Grotesk | 48px (`text-5xl`) | 700 | 1.0 (`leading-none`) | `-0.03em` (tight) | Section headings `<h2>`, `<h3>` |
| Label / Tag | Space Grotesk | 11px (`text-[0.7rem]`) | 700 | 1.2 | `0.2em` (wide, uppercase) | Small ALL-CAPS metadata tags, established-date badges |
| Body | Work Sans | 16px (`text-base` / `1rem`) | 500 | 1.5 | normal | Descriptions, body copy, nav links |

**Weights declared (2 only):**
- 500 (Work Sans medium) — body text
- 700 (Space Grotesk bold) — all display, heading, label uses

**Note on weight 700 vs CSS `font-weight: 900`:** Space Grotesk max available weight from `@fontsource/space-grotesk` is 700. The reference HTML uses `font-black` (CSS `font-weight: 900`). The browser font-matching algorithm will serve the 700 file when 900 is requested — this is acceptable behavior. The `@import` in `global.css` must be `@fontsource/space-grotesk/700.css`, never `/900.css` (that file does not exist and will throw a build error).

**Typography rules from DESIGN.md:**
- Do NOT use thin weights (300, 400). Brand lives in 500 and 700 only.
- Do NOT center-align everything. Left-aligned anchors are the editorial default.
- Tracking for display/heading: tightened (`-0.03em` to `-0.05em`).
- Tracking for label/tag elements: widened (`0.1em` to `0.3em`, uppercase).

**Source:** DESIGN.md §3 Typography, CONTEXT.md D-06, RESEARCH.md Pattern 3, juanaV1.html `font-black text-[10rem] leading-[0.85] tracking-tighter`.

## Color

Full palette declared in `@theme {}`. 60/30/10 split applied to dark-first surfaces.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-surface` | `#131313` | Base background — `<html>`, `<body>` background |
| Secondary (30%) | `--color-surface-low` / `--color-surface-high` | `#1a1a1a` / `#353534` | Content zones: cards sit on `surface-low`; elevated elements use `surface-high` |
| Accent (10%) | `--color-electric` | `#0055ff` | Reserved list — see below |
| Accent light | `--color-electric-light` | `#b6c4ff` | Decorative tint — Electric Blue at reduced contrast (selection highlight, subtle accents) |
| On-surface text | `--color-on-surface` | `#e5e2e1` | All body text, headings on dark backgrounds |

**Accent (`#0055ff`) is reserved for:**
- The `<body>` smoke-test background patch confirming Tailwind v4 tokens are live (Phase 1 success criterion 2)
- Any element that needs to confirm the Electric Blue CSS custom property is computed correctly in DevTools
- Primary CTA buttons (defined in later phases — spec here for token naming consistency)
- Active nav link underline (Phase 3 — but token must exist now)
- Marquee background (Phase 3)

**Accent must NOT be used on:** generic dividers, decorative borders, background fills of non-CTA elements, anything the user doesn't need to act on.

**No-Line Rule (from DESIGN.md §2):** 1px solid borders are prohibited for sectioning. Use background color shifts (`surface` → `surface-low` → `surface-high`) to define zones.

**Zero Roundedness Rule (from DESIGN.md §4):** `--radius-*: 0px` across all scales. Enforce in `@theme {}` by overriding every `--radius-{size}` token. Exception: `border-radius: 9999px` is only permissible for pill-shaped decorative elements if explicitly designed (reference HTML shows `"full": "9999px"` in config — not used in Phase 1).

**Dark mode:** Always-on. `<html class="dark">` is fixed. `@custom-variant dark (&:where(.dark, .dark *))` declared in `global.css`. No light mode toggle in Phase 1 or any v1 phase.

**Destructive color:** Not applicable in Phase 1. No destructive actions exist in this phase (scaffold + tokens + layout only). DESIGN.md references `error: #ffb4ab` for future input field error states — declare this as `--color-error: #ffb4ab` in `@theme {}` now so it's available in Phase 3 without revisiting global.css.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Error / Destructive | `--color-error` | `#ffb4ab` | Input field error states (Phase 3+). Declare now, use later. |

**Source:** CONTEXT.md D-06, DESIGN.md §2 Colors, REQUIREMENTS.md SETUP-04, juanaV1.html config block.

## Component Tokens (CSS vars, not Tailwind utilities)

These tokens do not map to Tailwind `@theme` namespaces — they live in `:root {}` and are consumed directly as CSS custom properties.

| Token | Value | Usage |
|-------|-------|-------|

**Declared in Phase 1** so Phase 2+ components can reference them without touching `global.css`.

## Interaction Patterns

Phase 1 has minimal interactions — it is scaffold and tokens. The contracts below are declared now to guide Phase 2+ implementations.

| Pattern | Spec | Source |
|---------|------|--------|
| Button hover | Instant color inversion — no transition duration. `transition-none` on all buttons. | DESIGN.md §5 Buttons |
| Nav link hover | Instant: `hover:bg-[electric] hover:text-white`, `transition-none` | juanaV1.html nav links |
| No divider lines | Background color shifts only — never `border-b` or `<hr>` for section separation | DESIGN.md §2 No-Line Rule |
| No rounded corners | `rounded-*` utilities all resolve to `0px` via `@theme` overrides | DESIGN.md §4 Zero Roundedness |
| Reduced motion | Declared as a requirement for Phase 3 marquee and Phase 4 GSAP — not implemented in Phase 1 | REQUIREMENTS.md MARQ-02, MOTN-02 |
